# 第25章 UI 架构设计

> 本文是对原文第 25 章的结构化重写。与前九章偏重 UGUI 底层渲染机制不同，本章讲的是如何在大型项目中组织 UI 工程代码——分层架构、管理调度、生命周期标准化。原文 3 节内容实为递进关系：分层 → 管理系统 → 生命周期，现按此逻辑整理，合并重复的概念说明。

---

## 概述

前几章关注的焦点是 UGUI 的渲染机制：顶点怎么生成、Batch 怎么合并、Shader 怎么渲染。但到项目落地时，还有一个更上层的问题——**代码怎么组织**。

如果几十上百个 UI 的创建、销毁、层级、事件绑定、数据刷新全部散落在各处，项目很快会变成"不敢改、改了就崩"的状态。

本章解决的三个问题：

```
① UI 分层与模块化  — 代码职责怎么分？
② UI 管理系统      — 谁负责创建/销毁/切换？
③ 生命周期标准化    — UI 从生到死的流程怎么统一？
```

## 25.1 UI 分层与模块化

### 25.1.1 四层架构

大型项目中，UI 代码通常按职责分为四层，每层只关心自己的事：

```
View（表现层）
  职责：显示 UI、响应用户输入
  特点：不包含业务逻辑，不直接操作数据源
  示例：
    public class LoginView : MonoBehaviour
    {
        public Button loginButton;
        public InputField accountInput, passwordInput;

        void Awake() => loginButton.onClick.AddListener(OnClickLogin);

        void OnClickLogin()
        {
            LoginController.Instance.Login(
                accountInput.text, passwordInput.text);
        }
    }

Controller（控制层）
  职责：接收 View 事件，调用 Model/Service，通知 View 更新
  特点：不直接操作 UI 组件，不负责数据存储
  示例：
    public class LoginController
    {
        public static LoginController Instance = new LoginController();
        private LoginView view;

        public void BindView(LoginView v) => view = v;

        public async void Login(string account, string password)
        {
            view.SetLoading(true);
            bool ok = await LoginService.Login(account, password);
            view.SetLoading(false);
            if (ok) { /* 登录成功 */ }
        }
    }

Model（数据层）
  职责：存储数据、提供访问接口、发送变更通知
  特点：与 UI 无关，可复用，可独立测试

Service（服务层）
  职责：网络请求、数据持久化、资源加载
  特点：与 UI 无关
```

**依赖方向：** `View → Controller → Model / Service`（单向依赖）

```
// ❌ 错误：Model 直接操作 UI
public class UserModel
{
    public void UpdateUI(Text t) => t.text = "xxx";
}

// ✅ 正确：Model 只负责数据，通过 Controller 转发给 View
public class UserModel
{
    public string Name { get; set; }
}
```

### 25.1.2 模块化组织

在分层基础上，将 UI 拆分为"功能独立的单元"，每个模块包含自己的 View、Controller、资源、配置：

```
InventoryModule/
├── InventoryView.cs
├── InventoryController.cs
├── InventoryItemView.cs
├── InventoryModel.cs
└── Prefabs/
```

模块之间通过**事件系统**或**接口**通信，避免直接引用：

```csharp
// 事件解耦
EventBus.Publish("OnLoginSuccess");

// 接口解耦
public interface ILoginModule
{
    void OnLoginSuccess();
}
```

## 25.2 UI 管理系统

### 25.2.1 分层结构（显示层级）

UI 显示层级用几个独立的 Canvas 或节点容器来组织：

```
UIRoot
├── Background Canvas  ← 底层主界面底图
├── Normal Canvas     ← 功能面板（背包、任务）
├── Popup Canvas      ← 弹窗（确认框、奖励）
├── Top Canvas        ← 顶层提示（Toast、飘字）
└── System Canvas     ← 加载界面、全局遮罩
```

每层独立 Canvas，互不影响——弹窗打开不会触发底层界面重建。

### 25.2.2 UI 管理系统的核心职责

```csharp
public class UIManager : MonoBehaviour
{
    private Dictionary<string, UIBase> uiDict = new Dictionary<string, UIBase>();

    public Transform normalLayer;

    public T OpenUI<T>(string name, GameObject prefab,
        object param = null, bool cache = true) where T : UIBase
    {
        if (uiDict.ContainsKey(name))
        {
            UIBase ui = uiDict[name];
            ui.OnOpen(param);
            ui.OnShow();
            ui.gameObject.SetActive(true);
            return ui as T;
        }

        GameObject go = Instantiate(prefab, normalLayer);
        T uiComp = go.GetComponent<T>();
        uiComp.OnCreate();
        uiComp.OnInit();
        uiComp.OnOpen(param);
        uiComp.OnShow();
        uiDict[name] = uiComp;
        return uiComp;
    }

    public void CloseUI(string name, bool destroy = false)
    {
        if (!uiDict.ContainsKey(name)) return;
        UIBase ui = uiDict[name];
        if (destroy)
        {
            ui.OnClose();
            ui.OnDestroyUI();
            Destroy(ui.gameObject);
            uiDict.Remove(name);
        }
        else
        {
            ui.OnHide();
            ui.gameObject.SetActive(false);
        }
    }
}
```

**三种缓存策略**：

| 策略 | 方式 | 适用场景 |
|------|------|---------|
| 完全销毁 | `Destroy()` | 低频 UI、占用内存大 |
| 隐藏缓存 | `SetActive(false)` | 高频 UI（背包、商城） |
| 对象池复用 | 池化管理 | 列表 Item、重复元素 |

### 25.2.3 UI 栈（Stack）机制

用于"返回上一界面"：

```csharp
private Stack<UIBase> uiStack = new Stack<UIBase>();

public void PushUI(UIBase ui)
{
    if (uiStack.Count > 0)
        uiStack.Peek().OnHide();  // 当前界面隐藏
    uiStack.Push(ui);
    ui.OnShow();
}

public void PopUI(bool destroy = false)
{
    if (uiStack.Count == 0) return;
    UIBase top = uiStack.Pop();
    if (destroy)
    {
        top.OnClose();
        Destroy(top.gameObject);
    }
    if (uiStack.Count > 0)
        uiStack.Peek().OnShow();  // 上一个界面恢复
}
```

## 25.3 界面生命周期标准化

### 25.3.1 UIBase 抽象类

所有 UI 继承同一个基类，生命周期由 UIManager 主动驱动，不依赖 Unity 的 `Start`/`OnEnable`：

```csharp
public abstract class UIBase : MonoBehaviour
{
    public virtual void OnCreate() { }     // 仅一次：实例化完成
    public virtual void OnInit() { }       // 仅一次：缓存引用、绑定事件
    public virtual void OnOpen(object param = null) { } // 每次打开：刷新数据
    public virtual void OnShow() { }       // 每次显示：动画、状态重置
    public virtual void OnHide() { }       // 隐藏时：暂停逻辑
    public virtual void OnClose() { }      // 关闭时：播放退出动画
    public virtual void OnDestroyUI() { }  // 销毁时：释放资源、解绑事件
}
```

### 25.3.2 执行时序

```
首次打开：Create → Init → Open(param) → Show
第二次打开（缓存）：Open(param) → Show
关闭（隐藏模式）：Hide
关闭（销毁模式）：Close → DestroyUI
```

**关键区分**：
- `OnInit()` 只执行一次——缓存引用、注册事件
- `OnOpen()` 每次打开都执行——设置数据、刷新 UI
- 千万不要在 `OnOpen` 里重复执行 `OnInit` 的逻辑

### 25.3.3 生命周期与动画

```csharp
public IEnumerator OpenWithAnimation(UIBase ui, object param)
{
    ui.OnOpen(param);
    ui.OnShow();
    yield return ui.GetComponent<Animator>().PlayAndWait("Open");
    // 动画完成后 UI 才可交互
}
public IEnumerator CloseWithAnimation(UIBase ui, bool destroy = true)
{
    yield return ui.GetComponent<Animator>().PlayAndWait("Close");
    if (destroy)
    {
        ui.OnClose();
        ui.OnDestroyUI();
        Destroy(ui.gameObject);
    }
}
```

### 25.3.4 常见问题

- **生命周期混乱**：不同 UI 用不同逻辑，维护困难 → 统一继承 UIBase
- **重复初始化**：在 `OnOpen` 里重复 `GetComponent` → 在 `OnInit` 缓存一次
- **依赖 Unity 生命周期**：用 `Start`/`OnEnable` 控制 UI 流程 → 由 UIManager 统一驱动
- **关闭逻辑不完整**：事件未解绑、资源未释放 → 统一在 `OnDestroyUI` 处理

## 本章总结

```
UI 架构三件套：

① 分层（解决职责问题）
   View → Controller → Model/Service
   单向依赖，每层只关心自己的事

② 管理系统（解决调度问题）
   UIManager 集中控制创建/销毁/层级/缓存
   UI 栈支持返回导航

③ 生命周期（解决时序问题）
   Create → Init → Open → Show → Hide → Close → DestroyUI
   统一基类，UIManager 驱动，不依赖 Unity 生命周期
```
