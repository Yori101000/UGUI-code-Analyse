# 第26章 UI 动画实现

> UGUI 没有内建 UI 动画系统，项目通常依赖 Animator、手写脚本或第三方 Tween 库。本章从工程角度出发，分析三种方案各自的适用场景、性能代价，并构建一套可复用的动画架构。

---

## 概述

UI 动画的本质是**属性变化**——位置、缩放、透明度、颜色等。这些变化会触发 UGUI 的 Dirty 标记机制，进而引发 Rebuild。所以动画的性能问题不来自"动"本身，而来自"动引发的重建成本"。

```
动画修改属性 → Dirty 标记 → CanvasUpdateRegistry → Rebuild → 更新 Mesh
```

---

## 26.1 三种动画方案对比

| 方案 | 驱动方式 | 适合 | 不适合 |
|------|---------|------|--------|
| **Animator** | 状态机 + 时间轴 | 固定流程动画（打开/关闭）、多状态切换 | 运行时动态计算、大量小 UI |
| **手写脚本** | Update/协程 | 简单一次性效果 | 复杂状态切换、维护成本高 |
| **Tween 系统** | 插值驱动 + 统一管理 | 通用 UI 动画、需要复用和控制 | 超复杂多状态切换 |

### 26.1.1 Animator

Animator 通过状态机驱动，适合结构固定的过渡动画：

```csharp
public class UIAnim : MonoBehaviour
{
    public Animator animator;

    public void PlayOpen()  => animator.SetTrigger("Open");
    public void PlayClose() => animator.SetTrigger("Close");
}
```

**优点**：可视化编辑，动画与代码解耦，美术可参与调参。
**缺点**：状态机易膨胀，运行时动态计算能力弱，每个 Animator 组件都有更新开销。

### 26.1.2 手写脚本动画

通过协程或 Update 直接控制属性：

```csharp
public IEnumerator ScaleAnim(RectTransform target, float duration)
{
    float t = 0;
    Vector3 start = Vector3.zero, end = Vector3.one;
    while (t < duration)
    {
        t += Time.deltaTime;
        target.localScale = Vector3.Lerp(start, end, t / duration);
        yield return null;
    }
    target.localScale = end;
}
```

**优点**：高度灵活，可动态计算。
**缺点**：不可复用，每个 UI 都手写相似的逻辑，维护成本高。

### 26.1.3 Tween 系统

Tween 是将"属性从 A 平滑到 B"这个过程标准化——指定起始值、目标值、时长、缓动函数，系统自动完成插值。

```
核心公式：value = Lerp(start, end, Ease(t / duration))
```

一个基础 Tween 引擎只有几十行代码：

```csharp
public class Tween
{
    public float duration;
    private float time;
    private bool isPlaying;
    private Action<float> onUpdate;
    private Action onComplete;

    public void Play(float dur, Action<float> update, Action complete = null)
    {
        duration = dur;
        onUpdate = update;
        onComplete = complete;
        time = 0;
        isPlaying = true;
    }

    public void Tick(float deltaTime)
    {
        if (!isPlaying) return;
        time += deltaTime;
        float t = Mathf.Clamp01(time / duration);
        onUpdate?.Invoke(t);
        if (t >= 1f)
        {
            isPlaying = false;
            onComplete?.Invoke();
        }
    }
}
```

通过 **TweenManager** 统一管理所有活动的 Tween：

```csharp
public class TweenManager : MonoBehaviour
{
    private List<Tween> tweens = new List<Tween>();

    public void Add(Tween t) => tweens.Add(t);

    void Update()
    {
        for (int i = tweens.Count - 1; i >= 0; i--)
            if (tweens[i].isPlaying) tweens[i].Tick(Time.deltaTime);
            else tweens.RemoveAt(i);
    }
}
```

**UI 常用动画封装成静态工具**：

```csharp
public static class TweenUtil
{
    public static Tween Move(RectTransform target, Vector3 start, Vector3 end, float dur)
    {
        Tween t = new Tween();
        t.Play(dur, p => target.anchoredPosition = Vector3.Lerp(start, end, p));
        return t;
    }

    public static Tween Fade(CanvasGroup group, float start, float end, float dur)
    {
        Tween t = new Tween();
        t.Play(dur, p => group.alpha = Mathf.Lerp(start, end, p));
        return t;
    }

    public static Tween Scale(RectTransform target, Vector3 start, Vector3 end, float dur)
    {
        Tween t = new Tween();
        t.Play(dur, p => target.localScale = Vector3.Lerp(start, end, p));
        return t;
    }
}
```

## 26.2 动画性能

动画的代价不在"变化"本身，而在变化触发的 Rebuild。

### 26.2.1 不同属性的代价

| 属性变化 | 触发什么 | 成本 |
|---------|---------|:---:|
| `localPosition` / `localScale` | Canvas 重新计算包围盒 | **低** |
| `anchoredPosition` | 同 localPosition（如果无 Layout 依赖） | **低** |
| `sizeDelta` | Layout Rebuild 递归计算 | **高** |
| `color` / `fillAmount` | Graphic Rebuild（顶点更新） | **中** |
| `alpha`（CanvasGroup） | 不触发 Rebuild（GPU blend 控制） | **极低** |
| 修改 text 内容 | 字体图集查询 + 顶点重建 | **中高** |
| LayoutGroup 下的属性 | Layout Rebuild 递归整棵子树 | **高** |

**最昂贵的做法**：在 LayoutGroup 的子节点上做 sizeDelta 动画 → 每帧触发全树 Layout Rebuild。

### 26.2.2 GPU 侧：Overdraw 与 DrawCall

- 透明度动画不会增加 DrawCall（不改材质，只改 CanvasGroup.alpha 或顶点颜色）
- 但在半透明动画过程中，重叠区域会产生 Overdraw（GPU 帧率受影响）
- 动画过程中突然切换材质（如 Sprite 替换）会打断 Batch

### 26.2.3 优化原则

```
✅ 优先改 localScale / localPosition / CanvasGroup.alpha
✅ 动画节点从 LayoutGroup 中脱离（放到同级但不受 Layout 控制）
✅ 高频动画集中到独立子 Canvas（隔离 Rebuild）
✅ 统一的 TweenManager 而不是每个 UI 各自起协程

❌ 避免在 LayoutGroup 内做尺寸动画
❌ 避免大量 UI 同时启动 Animator（每个组件都有 Update 开销）
❌ 避免动画过程中频繁替换 Sprite / 材质
```

## 26.3 动画与生命周期整合

动画应当与 UI 生命周期绑定，而不是散落在各个 MonoBehaviour 的 Update 中：

```csharp
[RequireComponent(typeof(UIAnimationController))]
public abstract class UIBase : MonoBehaviour
{
    protected UIAnimationController animController;

    protected virtual void Awake()
    {
        animController = GetComponent<UIAnimationController>();
    }

    // 由 UIManager 调用
    public virtual void OnOpen(object param = null)
    {
        animController?.PlayOpen();
    }

    public virtual void OnClose(Action onCompleted)
    {
        if (animController != null)
            animController.PlayClose(onCompleted);
        else
            onCompleted?.Invoke();
    }
}
```

**关闭流程带退出动画**：

```csharp
// UIManager 中
public void CloseUI(string name)
{
    if (!uiDict.ContainsKey(name)) return;
    UIBase ui = uiDict[name];
    ui.OnClose(() =>
    {
        Destroy(ui.gameObject);
        uiDict.Remove(name);
    });
}
```

确保动画播放完毕后再销毁 UI——避免退出动画被中途截断。

## 26.4 UIAnimationController 分层方案

将"动画描述"和"动画调度"分离：

**① Animation Component**——挂载在 UI 上，只描述"这个 UI 有什么动画"：

```csharp
public class UIAnimation : MonoBehaviour
{
    public enum Type { Scale, Fade, Move }

    public Type type;
    public float duration = 0.3f;
    public Vector3 startValue;
    public Vector3 endValue;

    private RectTransform rt;
    private CanvasGroup cg;

    void Awake()
    {
        rt = transform as RectTransform;
        cg = GetComponent<CanvasGroup>();
    }

    public Tween PlayForward()
    {
        return type switch
        {
            Type.Scale => TweenUtil.Scale(rt, startValue, endValue, duration),
            Type.Fade  => TweenUtil.Fade(cg, startValue.x, endValue.x, duration),
            Type.Move  => TweenUtil.Move(rt, startValue, endValue, duration),
            _ => null
        };
    }

    public Tween PlayBackward()
    {
        return type switch
        {
            Type.Scale => TweenUtil.Scale(rt, endValue, startValue, duration),
            Type.Fade  => TweenUtil.Fade(cg, endValue.x, startValue.x, duration),
            Type.Move  => TweenUtil.Move(rt, endValue, startValue, duration),
            _ => null
        };
    }
}
```

**② Animation Controller**——负责调度所有子动画，控制播放流程：

```csharp
public class UIAnimationController : MonoBehaviour
{
    private UIAnimation[] animations;

    void Awake() => animations = GetComponentsInChildren<UIAnimation>(true);

    public void PlayOpen()
    {
        foreach (var a in animations)
            TweenManager.Instance.Add(a.PlayForward());
    }

    public void PlayClose(Action onComplete)
    {
        int count = animations.Length;
        if (count == 0) { onComplete?.Invoke(); return; }

        foreach (var a in animations)
        {
            var t = a.PlayBackward();
            t.OnComplete(() =>
            {
                if (--count == 0) onComplete?.Invoke();
            });
            TweenManager.Instance.Add(t);
        }
    }
}
```

这样分层的好处：
- 设计师/策划可以在 Prefab 上配置 `UIAnimation` 组件，调参数不用改代码
- Controller 统一调度，不关心具体动画类型
- 可以随时在 Prefab 上添加/删除动画组件，不影响逻辑

## 本章总结

```
UI 动画三要素：

① 实现方式
   Animator → 适合固定流程动画（打开/关闭）
   手写脚本 → 适合简单一次性效果
   Tween    → 推荐的通用方案（标准化 + 可管理）

② 性能要点
   最便宜：localScale / position / CanvasGroup.alpha
   最贵：LayoutGroup 内的尺寸动画
   原则：从 Layout 中脱离、隔离到子 Canvas、统一管理 Tween

③ 工程整合
   UIAnimation（描述）→ UIAnimationController（调度）→ Tween（执行）
   与 UIBase 生命周期绑定（OnOpen 触发进入动画，OnClose 等待退出动画再销毁）
```
